## Reglas de Desarrollo
1. Código
Escribir código limpio y legible.
Usar nombres de variables descriptivos.
Evitar código innecesariamente complejo.
Comentar solo cuando aporte valor.
2. Estructura
Mantener una arquitectura clara.
Separar responsabilidades (componentes, lógica, servicios).
Reutilizar código cuando sea posible.
3. Estilo
Seguir consistencia en todo el proyecto.
Mantener formato uniforme (indentación, comillas, etc.)

4. siempre pensar en el responsive

5. cuando la tarea conlleva muchos cambios siempre crear un plan en español


## Lo que NO se debe hacer
No usar librerías innecesarias.
No complicar soluciones simples.
No asumir cosas sin contexto del proyecto.
No generar código sin explicar si es complejo.

## Forma de Responder

Ir directo al punto (sin rodeos).
Explicar primero la idea y luego el código.
Dar ejemplos prácticos.
Adaptarse al nivel del usuario (explicaciones claras, sin palabras complicadas).

## Flujo de Trabajo
Analizar problema
Proponer solución simple
Implementar código
Probar
Optimizar si es necesario
